# Edge Cases — Sub-Phase 1.6: Output Writer

> Covers: JSONL serialization, meta.json generation, per-source provenance files, and file system concerns.

---

## JSONL Writing

### EC-1.6.01 — Text Contains Newlines (Breaks JSONL Format)

| Aspect | Detail |
|--------|--------|
| **Trigger** | Review text contains literal `\n` characters |
| **Risk** | JSONL format requires one JSON object per line; embedded newlines break this |
| **Mitigation** | `json.dumps()` escapes newlines to `\\n` by default. Verify with `ensure_ascii=False` (to keep Unicode) but confirm newlines are escaped. Add a post-write validation that counts lines vs records. |

### EC-1.6.02 — Text Contains Characters That Break JSON

| Aspect | Detail |
|--------|--------|
| **Trigger** | Unescaped quotes, backslashes, or control characters in review text |
| **Risk** | Invalid JSON on that line |
| **Mitigation** | `json.dumps()` handles this correctly by default. Use `ensure_ascii=False` for readability. Never manually construct JSON strings. |

### EC-1.6.03 — Empty Record List (Zero Reviews)

| Aspect | Detail |
|--------|--------|
| **Trigger** | All records filtered out, or all sources failed |
| **Risk** | Empty `reviews_raw.jsonl` file |
| **Mitigation** | Write an empty file (0 bytes). In `meta.json`, set `"total": 0` and `"status": "empty"`. Log a warning. |

### EC-1.6.04 — Very Large Output File (>100 MB)

| Aspect | Detail |
|--------|--------|
| **Trigger** | Unlikely given caps (3,000 records max) but possible if text fields are very long |
| **Risk** | Slow writes; memory pressure |
| **Mitigation** | Write records one at a time in streaming mode (`for record in records: f.write(json.dumps(record) + '\n')`). Never build the entire file content as a single string. |

### EC-1.6.05 — File Encoding Mismatch

| Aspect | Detail |
|--------|--------|
| **Trigger** | File opened without specifying encoding on Windows |
| **Risk** | Windows defaults to `cp1252`; non-ASCII characters corrupted |
| **Mitigation** | Always open files with `encoding='utf-8'`: `open(path, 'w', encoding='utf-8')`. |

## meta.json

### EC-1.6.06 — `fetched_at` Timestamp Timezone Ambiguity

| Aspect | Detail |
|--------|--------|
| **Trigger** | `datetime.now()` returns local time without timezone info |
| **Risk** | Ambiguous timestamp in meta |
| **Mitigation** | Use `datetime.utcnow().isoformat() + 'Z'` or `datetime.now(timezone.utc).isoformat()`. Always UTC. |

### EC-1.6.07 — Counts Don't Add Up

| Aspect | Detail |
|--------|--------|
| **Trigger** | Sum of per-source counts ≠ `total` due to cross-source dedup |
| **Risk** | Confusing meta.json |
| **Mitigation** | Add explicit fields: `"total_before_cross_dedup"` (sum of per-source counts) and `"total"` (after cross-source dedup). Include `"cross_source_duplicates_removed"`. |

### EC-1.6.08 — `date_range` Doesn't Match Actual Data

| Aspect | Detail |
|--------|--------|
| **Trigger** | `date_range` in meta hardcodes the config window, but actual oldest review is newer |
| **Risk** | Misleading provenance |
| **Mitigation** | Report two date ranges: `"configured_range"` (from config) and `"actual_range"` (min/max dates from actual records). |

## Per-Source Provenance

### EC-1.6.09 — Raw Response Files Accumulate Over Runs

| Aspect | Detail |
|--------|--------|
| **Trigger** | Each run saves raw responses to `data/raw/{source}/`; files pile up |
| **Risk** | Disk usage grows; confusion about which files belong to which run |
| **Mitigation** | Organize by run timestamp: `data/raw/{source}/{run_timestamp}/`. Alternatively, overwrite raw files each run (simpler; raw data is ephemeral). Decision logged in `decisions.md`. |

### EC-1.6.10 — Per-Source meta.json Overwrites Previous Run

| Aspect | Detail |
|--------|--------|
| **Trigger** | New run overwrites `data/raw/play_store/meta.json` |
| **Risk** | Lose provenance of previous runs |
| **Mitigation** | If historical provenance is needed, use timestamped filenames (`meta_2026-06-20T18-30-00Z.json`). For Phase 1, overwriting is acceptable — each run produces a self-contained snapshot. |
