# Edge Cases — Sub-Phase 1.5: Pipeline Orchestrator

> Covers: Cross-source coordination, partial failure handling, output writing, and meta.json integrity.

---

## Source Failure Modes

### EC-1.5.01 — One Source Fails, Others Succeed

| Aspect | Detail |
|--------|--------|
| **Trigger** | Play Store fetcher throws an unrecoverable error; App Store and Reddit succeed |
| **Risk** | Incomplete dataset, but still partially useful |
| **Mitigation** | 1. Catch top-level exceptions per source. 2. Log the failure. 3. Continue with remaining sources. 4. In `meta.json`, set count to `0` for the failed source and add an `"errors"` entry. 5. Set `"status": "partial"`. |

### EC-1.5.02 — All Sources Fail

| Aspect | Detail |
|--------|--------|
| **Trigger** | Network outage, all APIs down, or configuration error |
| **Risk** | Empty output files |
| **Mitigation** | 1. Write an empty `reviews_raw.jsonl`. 2. Write `meta.json` with `"status": "failed"`, `"total": 0`, and all errors logged. 3. Exit with non-zero status code. 4. Do **not** overwrite a previously successful `reviews_raw.jsonl` — rename the failed output with a `.failed` suffix. |

### EC-1.5.03 — Source Returns Zero Records After Cleaning

| Aspect | Detail |
|--------|--------|
| **Trigger** | Fetcher succeeded but all records were dropped by Extractor/Cleaner (e.g., all outside date range) |
| **Risk** | Confusing meta.json — fetcher reports success but count is 0 |
| **Mitigation** | Distinguish in meta: `"raw_count": 1500, "clean_count": 0`. Add an alert: `"All records from play_store were dropped during cleaning."` |

## Cross-Source Issues

### EC-1.5.04 — Cross-Source Dedup Reduces Count Significantly

| Aspect | Detail |
|--------|--------|
| **Trigger** | Unexpectedly high overlap between sources (e.g., copypasted reviews) |
| **Risk** | Final total much lower than sum of per-source counts |
| **Mitigation** | Log: `"cross_source_duplicates_removed": 47`. Report both pre-dedup and post-dedup totals in meta. |

### EC-1.5.05 — Ordering Conflicts When Merging

| Aspect | Detail |
|--------|--------|
| **Trigger** | Records from different sources have different timezone handling; sorting by `date` produces unexpected order |
| **Risk** | `reviews_raw.jsonl` is not consistently sorted |
| **Mitigation** | Normalize all dates to UTC before sorting. Use `dateutil.parser.parse(date).astimezone(timezone.utc)` for comparison. |

## Output Integrity

### EC-1.5.06 — Disk Full During Write

| Aspect | Detail |
|--------|--------|
| **Trigger** | `reviews_raw.jsonl` write fails mid-stream due to disk space |
| **Risk** | Truncated/corrupted JSONL file |
| **Mitigation** | 1. Write to a temp file first (`reviews_raw.jsonl.tmp`). 2. Only rename to final name after successful write + flush. 3. If write fails, the previous file (if any) is preserved. |

### EC-1.5.07 — `meta.json` and `reviews_raw.jsonl` Out of Sync

| Aspect | Detail |
|--------|--------|
| **Trigger** | Pipeline crashes after writing JSONL but before writing meta |
| **Risk** | Meta reports stale counts or is missing entirely |
| **Mitigation** | Write meta **after** JSONL. Include a `"jsonl_sha256"` hash of the output file in meta so consumers can verify integrity. |

### EC-1.5.08 — Concurrent Pipeline Runs

| Aspect | Detail |
|--------|--------|
| **Trigger** | User accidentally starts two pipeline runs simultaneously |
| **Risk** | File corruption; interleaved writes |
| **Mitigation** | Use a lockfile (`data/output/.pipeline.lock`). If lock exists and the PID is still running, abort with an error message. Clean up stale locks if the PID is dead. |

## Configuration Issues

### EC-1.5.09 — Invalid Config Values

| Aspect | Detail |
|--------|--------|
| **Trigger** | `--months 0`, `--sources ""`, missing API endpoint URLs in config |
| **Risk** | Crash or unexpected behavior |
| **Mitigation** | Validate all config at startup before any fetching begins. Fail fast with a descriptive error message. |

### EC-1.5.10 — `--dry-run` Mode Edge Cases

| Aspect | Detail |
|--------|--------|
| **Trigger** | User runs `--dry-run` and expects to see what *would* happen |
| **Risk** | Accidentally making real HTTP calls in dry-run mode |
| **Mitigation** | In dry-run mode: print the config, list sources, show target URLs, estimate request counts. Make **zero** HTTP calls. Exit cleanly. |
