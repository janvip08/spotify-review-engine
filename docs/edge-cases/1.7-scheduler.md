# Edge Cases — Sub-Phase 1.7: Scheduler / Refresh

> **Phase 1 status:** Manual CLI trigger only. Scheduling is future work.
> These edge cases are documented for when automated refresh is implemented.

---

## Phase 1 (Manual Trigger)

### EC-1.7.01 — User Runs Pipeline Multiple Times in a Day

| Aspect | Detail |
|--------|--------|
| **Trigger** | Multiple manual `--run` invocations on the same day |
| **Risk** | Duplicate work; file overwrites |
| **Mitigation** | Default is overwrite mode — each run produces a fresh `reviews_raw.jsonl`. This is safe. If user wants to preserve previous runs, they should manually copy/rename output before re-running. |

### EC-1.7.02 — User Runs With Partial Sources

| Aspect | Detail |
|--------|--------|
| **Trigger** | `--sources reddit` (only Reddit) followed later by `--sources play_store,app_store` |
| **Risk** | Each run overwrites the full output file — Reddit data lost on second run |
| **Mitigation** | In partial mode, output to a source-specific file: `reviews_raw_reddit.jsonl`. A separate merge step (or full run) produces the combined file. Alternatively, warn the user that partial runs overwrite the combined output. |

## Future Scheduler (Automated Refresh)

### EC-1.7.03 — Scheduled Run Overlaps With a Running Pipeline

| Aspect | Detail |
|--------|--------|
| **Trigger** | Cron triggers a new run while the previous one is still fetching |
| **Risk** | Concurrent writes; corrupted output |
| **Mitigation** | Lockfile mechanism (see EC-1.5.08). If lock is held, skip this scheduled run and log `"skipped: previous run still in progress"`. |

### EC-1.7.04 — Append Mode Creates Growing File Without Bound

| Aspect | Detail |
|--------|--------|
| **Trigger** | Weekly refresh appends new reviews; file grows indefinitely |
| **Risk** | Performance degradation; stale old data dilutes fresh data |
| **Mitigation** | Enforce a rolling window: on each append run, also drop records older than 4 months. This keeps the dataset fresh. Log how many old records were pruned. |

### EC-1.7.05 — Dedup on Append Is Expensive

| Aspect | Detail |
|--------|--------|
| **Trigger** | Append mode requires loading all existing hashes into memory |
| **Risk** | Slow startup if `reviews_raw.jsonl` has grown to millions of lines |
| **Mitigation** | Maintain a sidecar hash index: `reviews_raw.hashes` (one SHA-256 per line, matching JSONL line numbers). Faster than re-hashing the full file. |

### EC-1.7.06 — Stale Data Detection

| Aspect | Detail |
|--------|--------|
| **Trigger** | `meta.json.fetched_at` is 30 days old; no refresh has run |
| **Risk** | Analysis based on outdated reviews |
| **Mitigation** | Any downstream consumer should check `fetched_at` and warn if `> 7 days` stale. The scheduler should log staleness alerts. In Phase 5 (RAG chatbot), display a banner: "Data last refreshed X days ago." |

### EC-1.7.07 — Source Endpoint Becomes Permanently Unavailable

| Aspect | Detail |
|--------|--------|
| **Trigger** | Reddit changes its JSON API; App Store RSS feed is deprecated |
| **Risk** | Scheduled runs silently fail for one source forever |
| **Mitigation** | 1. Track consecutive failures per source in a persistent counter file. 2. After 3 consecutive failed runs, escalate from WARNING to ERROR in logs. 3. Optionally send a notification (webhook/email) if configured. |

### EC-1.7.08 — Clock Skew on Scheduled Host

| Aspect | Detail |
|--------|--------|
| **Trigger** | Server clock is off by hours; `fetched_at` and date-range calculations are wrong |
| **Risk** | Reviews outside the 4-month window included or valid reviews excluded |
| **Mitigation** | Use NTP-synced time. In the pipeline, log the system clock at startup. If delta between system time and an HTTP `Date` header exceeds 5 minutes, log a warning. |

### EC-1.7.09 — Daylight Saving Time Transitions

| Aspect | Detail |
|--------|--------|
| **Trigger** | Pipeline runs during a DST transition; local time jumps forward/back |
| **Risk** | Date-range calculations off by 1 hour |
| **Mitigation** | All internal date calculations use UTC (`datetime.now(timezone.utc)`). Never use local time for comparisons or cutoffs. |
