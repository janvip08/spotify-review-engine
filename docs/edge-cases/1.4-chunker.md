# Edge Cases — Sub-Phase 1.4: Chunker

> **Phase 1 status:** Pass-through stub — no chunking is performed.
> These edge cases are documented now for Phase 2 readiness.

---

## Phase 1 (Current)

### EC-1.4.01 — Chunker Accidentally Mutates Records

| Aspect | Detail |
|--------|--------|
| **Trigger** | Even though the chunker is a pass-through, a code change accidentally modifies records |
| **Risk** | Data corruption |
| **Mitigation** | Unit test: assert `chunker.chunk(records) == records` (identity check). Use `copy.deepcopy` if there's any doubt. |

---

## Phase 2 (Future — Documented for Planning)

### EC-1.4.02 — Very Short Text (< Chunk Size)

| Aspect | Detail |
|--------|--------|
| **Trigger** | A review is 15 tokens; chunk size is 512 tokens |
| **Risk** | Creates a single tiny chunk |
| **Mitigation** | Return the text as a single chunk without splitting. Set `chunk_index = 0`, `chunk_total = 1`. |

### EC-1.4.03 — Very Long Text (> 10× Chunk Size)

| Aspect | Detail |
|--------|--------|
| **Trigger** | A Reddit post is 6,000 tokens |
| **Risk** | Creates many chunks; dilutes the metadata |
| **Mitigation** | Accept multiple chunks. Each inherits the parent's `source`, `date`, `url`, `rating`. Add `chunk_index` and `chunk_total` fields. Cap at 20 chunks per record (drop overflow). |

### EC-1.4.04 — Sentence Boundary Detection Fails

| Aspect | Detail |
|--------|--------|
| **Trigger** | Text has no punctuation (e.g., stream-of-consciousness review) |
| **Risk** | Chunker splits mid-word or produces uneven chunks |
| **Mitigation** | Fallback to character-based splitting with overlap when sentence tokenizer finds 0 sentence boundaries. |

### EC-1.4.05 — Text With Mixed Languages Mid-Sentence

| Aspect | Detail |
|--------|--------|
| **Trigger** | Review mixes English and another language ("esta app is terrible") |
| **Risk** | Sentence boundary detection confused |
| **Mitigation** | Use a language-agnostic splitter as fallback. The primary sentence tokenizer should be `nltk.sent_tokenize` with English model. |

### EC-1.4.06 — Overlap Exceeds Chunk Size

| Aspect | Detail |
|--------|--------|
| **Trigger** | Misconfiguration: `overlap=200`, `chunk_size=100` |
| **Risk** | Infinite loop or identical chunks |
| **Mitigation** | Validate at startup: `assert overlap < chunk_size`. Raise `ConfigError` if violated. |

### EC-1.4.07 — Chunk Metadata Orphaned From Parent

| Aspect | Detail |
|--------|--------|
| **Trigger** | Chunk loses reference to its parent record |
| **Risk** | Cannot trace chunk back to original review for citation |
| **Mitigation** | Every chunk inherits a `parent_id` field (the SHA-256 hash from Cleaner). This links chunks to the original `reviews_raw.jsonl` record. |
